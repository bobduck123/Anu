🔄 IF YOU NEED MANUAL MIGRATIONS LATER

  Set environment variable in Vercel:
    MIGRATION_TOKEN = your-secret-key

  Then call:
    curl -X POST https://anu-impact-service.vercel.app/api/migrations \
      -H "Content-Type: application/json" \
      -d '{"action": "deploy", "token": "your-secret-key"}'

❓ QUESTIONS?

  Q: Will this break anything?
  A: No. It just adds automatic migration running to your build. Safe.

  Q: Can I still deploy locally?
  A: Yes. Use "npm run falak:deploy:sandbox" as before.

  Q: What if deployment fails?
  A: Check Vercel logs. Run "git revert HEAD" and redeploy.

  Q: When do migrations run?
  A: During the build step, before your service goes live.
