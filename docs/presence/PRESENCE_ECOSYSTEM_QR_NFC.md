# Presence Ecosystem QR and NFC

## QR

The public QR route is:

`GET /api/presence/public/<slug>/qr`

It returns SVG generated by `qrcode==8.2`. The encoded value is the canonical public URL for the node.

If the dependency is unavailable in a development environment, the service logs a warning and falls back to a non-scanner placeholder so the route still responds. Production and pilot environments must install `qrcode==8.2`.

## Public UI

Public pages expose:

- share action with Web Share API fallback
- copy public URL
- vCard download
- QR display

Owner `/app/qr-nfc` exposes:

- canonical public URL
- copy/share/vCard controls
- QR image from the backend route
- NFC/source tag list
- create source tag flow

## NFC and Source Tracking

Supported public source forms:

- `/p/<slug>?source=nfc-card`
- `/p/<slug>?source=van-sticker`
- `/p/<slug>?nfc=<source_code>`

The public page records source hits through:

`POST /api/presence/public/<slug>/nfc-hit`

Anonymous source hits create analytics/interactions only. They do not create named contacts.

## Privacy Rule

Named connections require submitted details through an enquiry, quote request, or manual owner/admin entry.

Public APIs must not expose:

- relationship ledger
- named connections
- NFC tag internals
- quotes
- variations
- invoice support
- handovers

Owner app surfaces show NFC tags and analytics/source signals only. Staff control routes remain the place for full relationship-ledger review in alpha.

## Pilot Checklist

1. Open `/api/presence/public/<slug>/qr`.
2. Confirm SVG contains real QR path data.
3. Scan with a phone.
4. Confirm it resolves to `/p/<slug>`.
5. Visit `/p/<slug>?nfc=<source_code>`.
6. Confirm analytics/source hit is recorded without creating a named connection.
