# Presence Visual System

## The Cultural World System

A Presence is not a website.

A Presence is a cultural world interface.

It is where a person, practice, venue, artist, practitioner, organisation, or collective becomes publicly understandable. It is not only a page that displays content. It is a designed public surface that answers:

- Who is this?
- What world do they carry?
- What proof do they have?
- What can I do with them?
- How do they connect to a wider cultural system?

A Presence must function across six layers:

1. Identity - who/what appears
2. Atmosphere - what world is entered
3. Evidence - what proves the work
4. Opportunity - what can happen next
5. Memory - what is recorded or archived
6. Network - how this node can connect outward

A normal website is mostly identity plus contact.

A Presence is identity, atmosphere, evidence, opportunity, memory, and network in one designed surface.

That is the difference.

## The Core Doctrine

The platform should be scalable, but the outcome must feel authored.

The client should not feel they bought a template. They should feel:

> This is the first public surface that actually understands what I am.

The viewer should not feel they are browsing pages. They should feel:

> I have entered a specific world.

The studio should not be building from scratch every time. It should be assembling from reusable visual grammars:

- threshold
- gallery wall
- dossier
- studio wall
- care pathway
- public noticeboard
- archive river
- constellation field
- program wall
- ritual invitation
- physical-world QR/NFC bridge

The innovation is not custom websites faster.

The innovation is:

> Repeatable public worlds for people and organisations whose work is usually flattened by websites.

## The Presence Grammar

Do not design from sections. Design from verbs.

A generic website says:

- About
- Work
- Services
- Contact

A Presence should say, through its design:

- Enter
- Witness
- Understand
- Trust
- Ask
- Visit
- Support
- Refer
- Remember
- Map

Every template should have a primary verb.

Template verbs:

- Minimal Artist Portal = Enter
- Gallery Wall = Witness
- Editorial Portfolio = Understand
- Studio Practice = Trace
- Practitioner Presence = Be guided
- Venue / Collective = Gather

These are not skins. They are modes of encounter.

## Presence Quality Laws

### 1. The first screen must create a world

The first screen cannot simply introduce the person. It must establish the world. A good first screen can be sparse, quiet, image-led, typographic, or atmospheric, but it must create a specific threshold.

Bad:

> Hi, I am an artist based in Sydney.

Good:

> A dark projection field, a name, a slow image, one sentence, and an invitation to enter.

### 2. Proof must be designed, not listed

Proof is not just a list of works, credentials, services, testimonials, or events. Proof should appear as part of the world:

- works become objects
- collections become rooms
- services become pathways
- programs become public notices
- credentials become anchors
- statements become artefacts
- QR/NFC becomes a physical doorway

### 3. Contact must feel like continuation

The enquiry form should not feel like the end of a website. It should feel like the next natural action in the world.

For an artist:

> Request a private viewing.

For a practitioner:

> Begin a conversation.

For a venue:

> Propose a program.

For an editorial profile:

> Discuss a commission.

### 4. Every template needs a failure mode

A template is not ready until its failure mode is known. The failure mode is the shape the template collapses into when the media is weak, copy is thin, or layout decisions are generic.

### 5. The page must still work without spectacle

The core Presence must be beautiful without Three.js, without custom video, and without perfect media.

Atmosphere can elevate. It cannot be required for basic coherence.

## Six-Template Operating Doctrine

### 1. Minimal Artist Portal

- display_mode: `minimal_portal`
- world type: threshold
- primary verb: enter
- failure mode: black landing page

This is the most mythic template. It should feel like the doorway to an artist's world before it becomes a portfolio. The page should withhold slightly. It should not rush to explain. The visual system should create tension between mystery and access.

Expected structure:

1. Signal
2. Name
3. One-line world statement
4. Enter action
5. Revealed works
6. Statement
7. Private enquiry

The hero should be magnetic, not merely nice. The Enter button should feel like crossing a boundary, not clicking a marketing CTA.

### 2. Gallery Wall

- display_mode: `artist_gallery`
- world type: image field
- primary verb: witness
- failure mode: normal portfolio grid

This template should let the work speak before the biography does.

Images must have hierarchy. Some works should dominate. Some should sit like fragments. Some should behave like supporting objects. The page should feel arranged, not populated.

Expected structure:

1. Visual field
2. Featured work
3. Wall rhythm
4. Series / collections
5. Artist note
6. Print / commission / enquiry path

The grid must never feel database-generated.

### 3. Editorial Portfolio

- display_mode: `editorial_portfolio`
- world type: dossier
- primary verb: understand
- failure mode: resume / blog

This template is for people whose value is carried through judgement, writing, strategy, public record, cultural intelligence, or project evidence.

It should feel like a dossier assembled by a serious person. It can be typographic, numbered, annotated, issue-based, and evidence-led.

Expected structure:

1. Masthead
2. Thesis
3. Evidence files
4. Chapters / dossiers
5. Services as commissions
6. Contact as professional enquiry

The visitor should leave thinking:

> This person can think, assemble, explain, and deliver.

### 4. Studio Practice

- display_mode: `studio_practice`
- world type: studio wall
- primary verb: trace
- failure mode: beige artist profile

This template is for process.

It should treat unfinished work, fragments, failures, notes, sketches, studies, and material tests as public value. It should make the practice feel alive.

Expected structure:

1. Studio threshold
2. Practice statement
3. Material/process proof
4. Works and fragments
5. Collections as shelves or bodies of work
6. Enquiry as studio visit, commission, workshop, or collaboration

The template should feel warm and human, but not vague.

### 5. Practitioner Presence

- display_mode: `practitioner_profile`
- world type: care pathway
- primary verb: be guided
- failure mode: wellness coach funnel

This template is for healers, facilitators, educators, mentors, coaches, wellbeing practitioners, and cultural practitioners.

It should create trust through clarity, method, proof, and grounded invitation. It must avoid manipulative funnel language.

Expected structure:

1. Calm identity
2. Method / philosophy
3. Ways to work
4. Trust anchors
5. Location / service area
6. Booking or enquiry route
7. Supporting media

The page should make the visitor feel oriented, not sold to.

### 6. Venue / Collective Presence

- display_mode: `venue_profile`
- world type: public noticeboard
- primary verb: gather
- failure mode: venue brochure / event list

This template is for venues, collectives, studios, cultural organisations, small institutions, and community programs.

It should feel public, credible, warm, and institutionally useful. It should show a place, a mission, and a set of routes into participation.

Expected structure:

1. Public identity
2. Mission / about
3. Programs / services / spaces
4. Gallery / archive proof
5. Location / service area
6. Enquiry or proposal route
7. Directory / archive / support readiness signals where appropriate

The page should feel like a living cultural node, not a flyer.

## Scoring Rubric

Use this rubric for internal review, screenshot critique, and pilot acceptance.

| Score | Meaning | Description |
| --- | --- | --- |
| 0 | Broken | The page fails technically or visually. |
| 1 | Generic website | It works, but it could belong to anyone. |
| 2 | Styled website | It has visual style, but the world is not distinct. |
| 3 | Good portfolio | It presents identity and evidence clearly. |
| 4 | Distinct Presence | It feels authored, specific, and pilot-worthy. |
| 5 | Digital world | It becomes memorable as a public cultural surface. |

Minimum internal demo standard: 3.

Minimum invite-only pilot standard: 4.

Minimum public marketing standard: 4.5.

Flagship standard: 5.

## Operating Guardrails

- `frontend-next` is the canonical public pilot host.
- `presence-app` remains a standalone alpha and future public host until deliberately promoted.
- Do not port the six public templates into `presence-app` as routine maintenance.
- Do not add backend systems for visual doctrine.
- Do not add a seventh template to solve a styling problem.
- Do not make Three.js the default Presence language.
- Use atmosphere only when it clarifies identity, proof, memory, or opportunity.
