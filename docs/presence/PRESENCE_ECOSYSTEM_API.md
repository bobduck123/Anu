# Presence Ecosystem API

All routes preserve public/control separation. Public routes expose only public-safe serializers. Control routes require control-plane auth, RBAC scopes, and tenant/owner checks.

## Public API

- `GET /api/presence/public/<slug>`: returns published public/unlisted node data.
- `POST /api/presence/public/<slug>/enquiries`: creates an enquiry and named connection after submitted contact details.
- `POST /api/presence/public/<slug>/quote-request`: creates quote-request enquiry, named connection, draft quote, interaction, and analytics event.
- `POST /api/presence/public/<slug>/nfc-hit`: records anonymous NFC/QR interaction only; does not create a named connection.
- `GET /api/presence/public/<slug>/works/<work_id>`: returns public-safe node, visible work, and visible collection context.
- `GET /api/presence/public/<slug>/collections/<collection_id>`: returns public-safe node, visible collection, and visible works in that collection.
- `GET /api/presence/public/<slug>/vcard`: returns vCard.
- `GET /api/presence/public/<slug>/qr`: returns scanner-grade SVG QR generated by `qrcode==8.2`.
- `POST /api/presence/analytics/event`: captures public-safe analytics events.

## Owner API

Owner routes require participant/Supabase JWT auth and owned-node access. They are not host-gated control-plane routes.

- `GET /api/presence/owner/nodes`
- `GET/PATCH /api/presence/owner/nodes/<id>`
- `POST /api/presence/owner/nodes/<id>/publish`
- `POST /api/presence/owner/nodes/<id>/unpublish`
- `GET/POST /api/presence/owner/nodes/<id>/works`
- `PATCH/DELETE /api/presence/owner/works/<id>`
- `GET/POST /api/presence/owner/nodes/<id>/collections`
- `PATCH/DELETE /api/presence/owner/collections/<id>`
- `GET/POST /api/presence/owner/nodes/<id>/services`
