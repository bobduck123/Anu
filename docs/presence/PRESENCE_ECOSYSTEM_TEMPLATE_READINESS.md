# Presence Ecosystem Template Readiness

## Launch Position

Presence is creative-portfolio-led for invite-only pilots. The current platform supports the required portfolio structures and includes dedicated public front-end treatments for the six pilot templates.

This audit is the pilot gate for the six templates that matter before broader professional, tradie, and institutional modules are revisited.

## 1. Minimal Artist Portal

Required sections:

- landing title, subtitle, background image, and Enter gateway
- artist identity header
- practice statement
- selected works
- contact/enquiry
- QR, share, and vCard

Ideal media:

- one atmospheric landing image
- artist portrait or studio image
- 3 to 6 selected work images

Demo data needed:

- artist name, concise portal subtitle, practice statement, one collection, selected works, enquiry copy

Current support:

- Supported by `minimal_portal`, landing fields, statements, works, collections, public enquiry, `/app/presence`, and a sparse full-screen public portal treatment.

Remaining:

- real pilot image set and copy
- final screenshot QA on mobile and desktop

## 2. Gallery Wall

## Pilot Readiness Decision

All six pilot templates are structurally and visually supported by the current Presence alpha.

The remaining work before polished public launch is:

- real pilot media and copy
- final screenshot QA with selected pilot content
- hosted smoke run once credentials are available

No additional backend systems are required for the invite-only creative portfolio pilot.
