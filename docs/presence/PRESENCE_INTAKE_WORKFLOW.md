# Presence Intake Workflow

## The Presence Assembly Process

The intake is not a form.

It is a world assembly process.

Its job is to transform raw client material into a structured public world without making each build an endless bespoke project.

The output should be:

1. Presence Brief
2. World Direction
3. Content Pack
4. Evidence Map
5. Opportunity Routes
6. Media Plan
7. Launch Checklist

## Stage 1 - Public Signal

Purpose: define what appears first.

Prompt questions:

- What should people call this presence?
- What are you known for?
- What are you becoming known for?
- What should someone understand in 10 seconds?
- What should they feel before reading?
- What opportunity should this attract?

Output block:

```txt
Public Signal:
Name:
Role:
Location:
One-line world statement:
Primary CTA:
Template candidate:
```

## Stage 2 - World Selection

Purpose: select the spatial mechanism.

World options:

- threshold
- gallery wall
- public dossier
- studio wall
- care pathway
- public noticeboard
- archive river
- constellation
- object field
- venue room

Prompt questions:

- Is this more like a room, wall, archive, threshold, pathway, or noticeboard?
- Should the world feel sparse or dense?
- Should it feel warm or severe?
- Should it feel intimate or public?
- What visual references matter?
- What visual references are banned?

Output block:

```txt

Template:
Mechanism:
Mood:
Banned vibes:
Inspiration:
```

Per work:

## Stage 7 - Physical-World Bridge

Purpose: connect the Presence to real-world encounters.

QR/NFC contexts:

- business card
- NFC tag
- artwork label
- studio wall
- exhibition card
- venue entrance
- flyer
- poster
- event badge
- sponsor pack
- market stall
- workshop table

Prompt questions:

- Where will people encounter this in the world?
- Should the QR route to the full presence, a work, collection, or enquiry?
- What should the NFC tag say?
- Is this anonymous scan tracking or named enquiry capture?

Output block:

```txt
Physical Bridge Plan:
Default QR:
Special QR routes:
NFC label:
Source tags:
Named capture rules:
```

Named capture rule: anonymous scans can be counted, but named contacts require submitted details or manual owner/admin entry.

## Stage 8 - Launch Readiness

Purpose: prevent weak launches.

Minimum launch requirements:

- identity complete
- world mechanism selected
- one strong hero image
- short bio complete
- statement complete
- at least 3 works or proof items
- at least 1 collection or body of work
- primary opportunity route set
- enquiry tested
- QR tested
- mobile screenshot reviewed
- desktop screenshot reviewed
- no placeholder copy
- no broken media
- public/private visibility checked

Output block:

```txt
Launch Decision:
Ready:
Needs content:
Needs media:
Needs copy:
Needs design polish:
Not safe to publish:
```

## Intake Outputs

Each pilot should leave intake with these deliverables:

1. Presence Brief - the public identity and purpose.
2. World Direction - selected mechanism, template, mood, and banned vibes.
3. Content Pack - copy, statements, works, collections, services, and CTAs.
4. Evidence Map - visual proof, trust proof, public proof, opportunity proof, and private proof.
5. Opportunity Routes - what the public should do next and where the enquiry goes.
6. Media Plan - hero image, supporting images, missing images, hosted URL plan.
7. Launch Checklist - pass/fail decision for publication.

## Template Fit Questions

Use these to prevent weak template selection:

- If the first action is Enter, consider Minimal Artist Portal.
- If the first action is Witness, consider Gallery Wall.
- If the first action is Understand, consider Editorial Portfolio.
- If the first action is Trace, consider Studio Practice.
- If the first action is Be guided, consider Practitioner Presence.
- If the first action is Gather, consider Venue / Collective Presence.

The template choice should follow the world mechanism, not the client's job title.
