# Canvas Options Map

## Kimi Options Reviewed

| Candidate concept | Decision |
| --- | --- |
| Canvas shell, selection, overlay, toolbar, desktop/mobile inspector | Reimplemented against the real editor route and nested draft lifecycle |
| Inline text editor | Reimplemented in visible Canvas surfaces |
| Asset picker and alt text | Adapted to existing/attached/live images with safe advanced link |
| Work-wall reorder | Adapted as constrained reorder with move buttons |
| Mood, motion, readiness, pilot banner | Adapted to wired controls and owner language |
| Candidate `/canvas` route | Rejected; one canonical Studio editor surface retained |
| DOM scanning / mutation observation | Rejected; typed registry is authoritative |
| Backend/migration/token files | Rejected entirely |

## Integrated Options

| Owner option | Draft mapping | Public renderer status |
| --- | --- | --- |
| Core visible words | Nested scene/content/enquiry sections | GGM published renderer wired |
| Text size, font mood, colour, alignment, emphasis | Controlled `style_dna.element_styles` tokens | GGM published renderer wired; hero overlay colour intentionally constrained for readability |
| Paper / Ink / Warm / Liquid moods | Controlled palette and safe motion tokens | GGM published renderer wired |
| Still / Gentle / Living motion | Controlled motion tokens; heavy off and reduced-motion retained | GGM published renderer wired |
| Change image / alt text | Nested hero or work asset records | GGM published renderer uses published assets |
| Move work | Nested works mirrors plus wall order | GGM published renderer uses ordered works |
| Needs attention | Existing readiness report targeted to registry IDs | Owner Canvas action only |

## Deferred Or Rejected Options

| Option | Status | Reason |
| --- | --- | --- |
| Raw CSS controls | Rejected | Unsafe and unsuitable owner UX |
| Heavy / immersive motion in pilot | Hidden | Requires explicit later safety review |
| Freeform drag-anywhere layout | Rejected for v1 | Mobile/layout safety risk |
| Device upload | Deferred | No draft-scoped endpoint |
| Crop / focal point | Deferred | No real implementation endpoint |
| Full style controls for non-GGM rooms | Deferred | Must first be renderer-visible |

## Release Posture

- **Pilot-safe now:** GGM Canvas selection, copy, controlled style/mood/motion, existing-image choice, alt text, constrained order, preview, explicit publish.
- **Needed for paid pilot:** draft upload/crop/focal workflow, hosted proof, prepared support/runbook, required room renderer coverage.
- **Needed for public self-serve:** broad renderer support, robust uploads/media management, deeper accessibility/usability testing, and operational monitoring.
