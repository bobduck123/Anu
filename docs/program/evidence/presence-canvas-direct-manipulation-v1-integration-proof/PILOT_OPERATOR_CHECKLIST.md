# Pilot Operator Checklist

## Onboard The First User

1. Confirm the pilot room uses the wired GGM renderer and has an owner account with Studio access.
2. Prepare stable public image assets and descriptive alt text before onboarding.
3. Open the owner Studio route and confirm Canvas appears by default with the pilot banner.
4. If existing live images are shown as needing import, choose **Bring in live images** before asking the owner to move or change works.
5. Walk the owner through edit, preview, and **Open room to visitors** once; require them to perform the second pass themselves.

## Emad Must Prepare

- Owner identity/access and room assignment.
- Stable existing image set suitable for the pilot; no expiring asset links where avoidable.
- Enquiry posture, RoomKey configuration, and any staff-only settings in Advanced controls.
- Hosted smoke check of owner editor, private preview, public room, and RoomKey before a real visitor launch.

## The User Can Edit Alone

- Room title/caption, statement, practice title, biography, process notes, invitation title/body/button, and work titles/captions.
- Text style presets and wired mood/motion presets.
- Existing room/attached/live-room image choice and alt text.
- Work-wall order using the Canvas move controls.
- Needs-attention repairs exposed on Canvas.
- Draft preview and intentional open-to-visitors publishing.

## Emad May Still Need To Do

- Add genuinely new imagery until draft-scoped device upload exists.
- Crop, focal positioning, and image preparation.
- Configure advanced RoomKey/enquiry details or restore earlier versions.
- Run hosted verification and investigate any renderer type outside GGM.

## Known Risk Moments

- Owner selects an expiring signed image link instead of a stable room image.
- Owner expects device upload, crop, or focal controls that are not yet present.
- Owner opens the room without reviewing private preview on mobile.
- A non-GGM room is enrolled before renderer-visible controls are wired.

## Go / No-Go

- [ ] Owner access is confirmed and non-owner access is denied.
- [ ] Canvas opens by default and Advanced controls are not required for normal edits.
- [ ] All visible images have accurate alt text.
- [ ] Owner has reviewed desktop and mobile private preview.
- [ ] Public room remains unchanged until explicit confirmation.
- [ ] After confirmation, public room and RoomKey expose the published result only.
- [ ] Image assets are stable, public, and not expiring links.
- [ ] Hosted smoke verification has been completed for a real pilot.

No-go if any unchecked item affects the specific pilot launch.

## Post-Pilot Questions

- Which visible item did you first try to change, and did Canvas make that obvious?
- Did you ever feel forced into Advanced controls for an ordinary edit?
- Was changing an image understandable without staff explanation?
- Did the needs-attention guidance make the repair clear?
- Did preview and Open to visitors feel safely distinct?
- What image upload, crop, or layout capability did you expect next?
