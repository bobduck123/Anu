# Pilot Operator Checklist

## Onboard the First User

1. Prepare a GGM room with approved published copy, existing image choices, meaningful alt text prompts, and at least two works for reorder testing.
2. Give the owner the Studio sign-in path and explain: Canvas edits the Draft room; only Open to visitors changes the Live room.
3. Walk through one title edit, one mood/font choice, one image alt-text improvement, one work move, preview, and deliberate publish.
4. Keep Advanced controls out of the walkthrough unless a known deferred task requires operator intervention.

## Emad Prepares

- Approved room and owner access.
- Existing image library; upload is not part of this pass.
- Starting public screenshots and copy record.
- A rollback/earlier-version decision before publishing.
- A second browser/session for public boundary checks.
- Hosted smoke evidence following `HOSTED_SMOKE_STEPS.md`.

## The Owner Can Edit Alone In The Validated Path

- Visible room title and supported text in Canvas.
- Safe text style controls already present in Canvas.
- Curated font choice and exposed palette colours.
- Paper Gallery or Ink Room mood pack.
- Existing room image choice and alt text.
- Work-wall order.
- Readiness fixes already provided by Canvas.
- Full draft preview and deliberate Open to visitors confirmation.

## Emad May Still Need To Do

- Add new source images, crop or set focal points.
- Create new room blocks or additional gallery layouts.
- Edit RoomKey-specific copy through advanced/operator workflows.
- Support non-GGM rooms or fixed GGM chrome not exposed in Canvas.
- Supervise the first hosted publish and rollback decision.

## Risk Moments

- An owner expects an uploaded image tool that is not present.
- An owner expects drag-anywhere composition rather than constrained work ordering.
- A signed CDN image expires after selection.
- A high-motion draft is expected to be immersive while pilot safety keeps it comfortable.
- A hosted room exposes a parity issue not reproduced by local fixture data.

## Go / No-Go

- [ ] Hosted draft is not visible on the anonymous public room before publish.
- [ ] Full preview matches all intended draft edits.
- [ ] Public room matches those edits only after explicit publish.
- [ ] RoomKey exposes published content only.
- [ ] Mobile edit and work move fallback are usable on a real device.
- [ ] No broken image or missing required alt text remains.
- [ ] Owner understands what is deferred and can operate without Advanced controls for normal edits.
- [ ] Rollback/earlier-version path is understood before first public change.

## Post-Pilot Feedback Questions

- Did the room feel like the thing you were editing, or like a separate control screen?
- Which visible item did you try to change but could not?
- Did Draft room versus Live room behavior feel clear before publishing?
- Were font, mood, and image choices enough for your first edit?
- Did moving works feel reliable on phone and desktop?
- What required operator help that you expected to do yourself?
