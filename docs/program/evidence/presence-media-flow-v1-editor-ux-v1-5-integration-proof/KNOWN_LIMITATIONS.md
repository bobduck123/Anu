# Known Limitations

- **Upload:** Not available in Canvas Images mode. A legacy node-level uploader exists, but it is not a verified draft-room upload path.
- **Crop:** Not implemented.
- **Focal point:** Not implemented because draft and published renderer parity has not been proven for a focal-point field.
- **New image provisioning:** For this managed V1A pilot, an operator still prepares or attaches new images.
- **Image sources:** Media drawer exposes currently supported room/live candidate images; it does not create a full persistent media library.
- **Advanced editor:** Retained for operators and deep controls, and therefore still contains technical language after the owner deliberately opens Advanced.
- **GGM coverage:** The richest Canvas direct-manipulation/render parity remains GGM-oriented; non-GGM regressions are protected only to existing coverage.
- **Mobile:** Existing Canvas mobile handling remains; this pass did not add a new Media drawer mobile acceptance sweep beyond responsive rendering.
- **Hosted proof:** Not run for this new UX/Media change. Deployment and owner smoke are still required.
- **Paid pilots/public self-serve:** Not ready. Secure draft-scoped upload, crop/focal support or explicit deferral acceptance, hosted owner testing, and broader usability evidence remain required.
