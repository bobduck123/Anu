## Style Grammar Extracted

Reusable Presence grammar:

- full-viewport threshold first
- image-led entry ritual
- black field with gold mark/action language
- minimal chrome
- Enter transition into gallery space
- image field with selectable thumbnails
- large focused image stage
- low-profile Prev/Next controls
- small public art labels rather than card captions
- practice/about section from public-safe text objects
- mobile-first stacked gallery flow
- restrained hover/focus states
- reduced-motion support through existing public CSS guard

Not copied:

- no iframe
- no static one-off bbbvision page
- no hardcoded bbbvision content in the generic renderer
- no CodePen reset dependency
- no full canvas/glitch engine port
- no random private asset paths
- no fake upload/crop/storage/library features

Future model support:

- explicit threshold/landing image sequence selection
- explicit gallery orbit order
- real bbbvision asset migration and media optimization
- route-level Enter state if Presence later supports split landing/gallery public routes
- optional canvas/motion layer if accessibility and performance can be proven

## Presence Integration Path

Existing Presence mechanisms audited:

- Studio V2 public style selection already persists as `style_dna.studio_v2.publicStylePreset`.
- V2 public renderer activation remains gated by renderer key and pilot eligibility.
- Owner Studio reads use `/api/presence/owner/nodes/:id`.
- Editor save/preview/publish use `/api/presence/owner/rooms/:id/editor/*`.
- Existing owner node creation route creates authenticated draft/private nodes via `POST /api/presence/owner/nodes`.
- Backend owner-node list scopes nodes to `owner_user_id`, with platform-admin override.

Hosted owner index check:

- authenticated Studio index loaded successfully
- owner node list request returned `200`
- no bbbvision/BBB room was visible in the owner Studio index

Decision:

- This pass does not create or mutate a hosted bbbvision room.
- The correct hosted path is a controlled create/seed/assign pass using existing owner/admin routes, then conversion to Studio V2 with this preset selected.
- Local/mock tests prove the editable pilot path through normal Studio V2 draft/save/publish without hosted mutation.
