# DAILY_OPERATOR.md — Hyperdrive Rhythm

## Purpose

Keep execution moving without letting Pro/agents become another abstraction layer.

## Daily command pass

Run at the start of each work block.

```text
Today's date:
Energy/stability check:
Top 3 outcomes:
1.
2.
3.

Must ship/capture today:
Agent tasks:
Human decisions:
Blocked:
Do not touch today:
End-of-day proof target:
```

## Daily lanes

### Lane 1 — Presence proof

At least one of:
- migrated section;
- screenshot;
- demo route;
- case study update;
- launch page improvement;
- QA fix.

### Lane 2 — Revenue

At least one of:
- lead list;
- outreach draft;
- message sent;
- audit delivered;
- proposal drafted;
- follow-up sent;
- invoice/payment task.

### Lane 3 — Systems/proof

At least one of:
- canon update;
- Profiler/FBi proof note;
- workflow template;
- no-merge review;
- delivery checklist.

## End-of-day review

```text
Shipped:
Proof captured:
Revenue movement:
Decisions made:
Blocked:
Tomorrow's first task:
Canon updates needed:
```

## Weekly rhythm

### Monday

- choose weekly top 3;
- refresh proof/revenue boards;
- pick migration target;
- create tickets.

### Tuesday–Thursday

- agent execution;
- review/merge;
- client/revenue work;
- case study capture.

### Friday

- proof library update;
- revenue pipeline review;
- launch readiness score;
- next week plan.

## Anti-drift rule

If a session produces only clarity and no artefact, it is incomplete.

Acceptable artefacts:
- PR;
- ticket;
- case study;
- audit;
- proposal;
- screenshot;
- outreach list;
- launch page copy;
- canon update;
- blocker report with next action.
