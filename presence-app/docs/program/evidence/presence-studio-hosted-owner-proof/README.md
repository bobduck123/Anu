# Presence Studio hosted owner proof

## Purpose

This evidence pack records the hosted owner-bound Studio V2 proof gate. It exists to separate truthful hosted owner evidence from local Room 11-shaped fixture evidence.

## Scope

This pass is read-only by default.

It may verify:

- hosted owner sign-in;
- owner-bound backend read access;
- hosted Studio V2 editor rendering;
- layout-composition controls in the hosted editor;
- hosted private preview rendering;
- anonymous public route behaviour for the configured slug;
- absence of publish or owner mutation requests during the proof.

It must not:

- publish;
- mutate public state;
- call mutation-capable lifecycle specs;
- expose secrets;
- claim GGM is public launch material;
- treat fallback/frontend visual copy as system-native migration proof.

## Redacted env/config checklist

Real values are kept outside git in `.env.hosted-owner-proof.local` or in the local shell process.

Required keys:

- `PRESENCE_HOSTED_SMOKE`
- `PRESENCE_E2E_BASE_URL`
- `PRESENCE_E2E_API_URL`
- `PRESENCE_E2E_OWNER_EMAIL`
- `PRESENCE_E2E_OWNER_PASSWORD`
- `PRESENCE_STUDIO_V2_HOSTED_PILOT_ROOM_ID`
- `PRESENCE_STUDIO_V2_HOSTED_PILOT_SLUG`

Optional keys:

- `PRESENCE_STUDIO_V2_HOSTED_LEGACY_SLUG`
- `PRESENCE_STUDIO_V2_HOSTED_LEGACY_ROOM_ID`
- `PRESENCE_HOSTED_DRAFT_WRITE_ENABLED`

For this read-only proof, `PRESENCE_HOSTED_DRAFT_WRITE_ENABLED` must be unset or `0`.

## Known limits

- This proof does not perform draft write/revert.
- This proof does not publish.
- This proof does not prove migration completeness for GGM or BBB.
- This proof does not repair owner policy, control-plane credentials, or backend configuration.
- Public-route checks are route-render checks only; they do not intentionally call analytics-writing public-detail APIs directly.
