# BBB Vision hosted Studio V2 generalisation proof

## Purpose

This evidence pack records the hosted owner-bound BBB Vision Studio V2 generalisation proof for room `29 / bbbvision`.

It verifies that BBB can enter Studio V2 on a hosted frontend, render the environmental/layout-composition editor, load private preview, and leave anonymous public routes unchanged.

## Scope

This pass is read-only.

It may verify:

- hosted owner sign-in;
- owner-bound backend read access;
- BBB room identity through owner detail/editor reads;
- hosted Studio V2 editor rendering;
- layout-composition controls in the hosted editor;
- hosted private preview rendering;
- anonymous public route behaviour before and after proof;
- absence of publish or owner mutation requests during the proof.

It must not:

- publish BBB;
- mutate public state;
- write or revert drafts;
- call mutation-capable lifecycle specs;
- expose secrets;
- claim BBB is launched or publication-ready.

## Current run result

Passed as a production-hosted read-only proof on 2026-07-21.

The branch Preview deployment existed, but anonymous access to `/auth/sign-in` was intercepted by Vercel deployment protection before the Presence app loaded. Per human instruction, the proof was rerun against the regular production frontend.

Confirmed:

- hosted owner authentication reached Studio;
- owner-scoped backend list, detail, editor overview, and draft reads returned 200;
- owner API identity guard matched room `29` and slug `bbbvision`;
- hosted Studio V2 editor loaded for room `29`;
- exact BBB title `bbb.vision` rendered in the Studio V2 editor and private preview;
- layout-composition controls rendered in the hosted editor;
- environmental DOM room/editor view rendered;
- private preview loaded with no editor instrumentation;
- anonymous `/p/bbbvision` returned 200 before and after proof;
- anonymous `/presence/bbbvision` returned 200 before and after proof;
- public route statuses were unchanged by the proof;
- no publish request was triggered;
- no draft save/revert was attempted;
- no backend mutation was attempted beyond the existing safe private-preview generation request.

The private preview page uses the existing safe preview-generation endpoint, `POST /api/presence/owner/rooms/29/editor/preview`. The frontend client marks this as `ownerReadFetch` with `safeEnsurePost`; it does not publish or overwrite owner content.
