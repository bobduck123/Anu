# BBB Vision hosted Studio V2 generalisation proof

## Purpose

This evidence pack records the hosted owner-bound BBB Vision Studio V2 generalisation proof for room `29 / bbbvision`.

It verifies that BBB can enter Studio V2 on a hosted frontend, render the environmental/layout-composition editor, load private preview, and leave anonymous public routes unchanged.

## Scope

The initial hosted proof was read-only. The follow-up hosted draft proof was separately gated by
`PRESENCE_HOSTED_DRAFT_WRITE_ENABLED=1` and approved only for one reversible private draft mutation
followed by exact draft restoration.

The read-only pass may verify:

- hosted owner sign-in;
- owner-bound backend read access;
- BBB room identity through owner detail/editor reads;
- hosted Studio V2 editor rendering;
- layout-composition controls in the hosted editor;
- hosted private preview rendering;
- anonymous public route behaviour before and after proof;
- absence of publish or owner mutation requests during the proof.

It must not:

- publish BBB;
- mutate public state;
- call mutation-capable lifecycle specs;
- expose secrets;
- claim BBB is launched or publication-ready.

The gated draft write/revert pass may additionally:

- capture the original editable draft payload;
- change one narrow Studio V2 object-placement draft field;
- save the private draft;
- reload Studio and confirm persistence;
- load private preview from the temporary draft;
- restore the exact captured editable draft payload;
- reload Studio and confirm the revert;
- confirm anonymous public routes remain unchanged.

## Current run result

Passed as a production-hosted read-only proof on 2026-07-21.

The branch Preview deployment existed, but anonymous access to `/auth/sign-in` was intercepted by Vercel deployment protection before the Presence app loaded. Per human instruction, the proof was rerun against the regular production frontend.

Confirmed:

- hosted owner authentication reached Studio;
- owner-scoped backend list, detail, editor overview, and draft reads returned 200;
- owner API identity guard matched room `29` and slug `bbbvision`;
- hosted Studio V2 editor loaded for room `29`;
- exact BBB title `bbb.vision` rendered in the Studio V2 editor and private preview;
- layout-composition controls rendered in the hosted editor;
- environmental DOM room/editor view rendered;
- private preview loaded with no editor instrumentation;
- anonymous `/p/bbbvision` returned 200 before and after proof;
- anonymous `/presence/bbbvision` returned 200 before and after proof;
- public route statuses were unchanged by the proof;
- no publish request was triggered;
- no draft save/revert was attempted;
- no backend mutation was attempted beyond the existing safe private-preview generation request.

The private preview page uses the existing safe preview-generation endpoint, `POST /api/presence/owner/rooms/29/editor/preview`. The frontend client marks this as `ownerReadFetch` with `safeEnsurePost`; it does not publish or overwrite owner content.

## Hosted draft write/reload/revert result

Passed as a production-hosted gated draft proof on 2026-07-21.

Confirmed:

- `PRESENCE_HOSTED_DRAFT_WRITE_ENABLED=1` was required for the draft mutation spec;
- the original editable BBB draft payload was captured and normalized before mutation;
- the only intentionally changed draft field was `scene_config.studio_v2.objectState[object].chamberId`;
- the original and temporary chamber values were captured but not recorded in evidence;
- the private draft save returned 200;
- Studio reload confirmed the temporary draft value persisted;
- private preview loaded from the temporary draft after mutation;
- the exact captured editable draft payload was restored;
- Studio reload confirmed the normalized editable draft matched the original after revert;
- anonymous `/p/bbbvision` returned 200 before mutation and 200 after revert;
- anonymous `/presence/bbbvision` returned 200 before mutation and 200 after revert;
- no publish request occurred;
- no public status, slug, title, domain, backend auth, control-plane, tenant, route, or deployment configuration change was attempted.

This proves a narrow owner-bound BBB draft write/reload/revert loop only. It does not prove BBB public launch readiness and does not approve publication.

## Known limits

- This proof does not publish BBB.
- This proof does not prove BBB public launch readiness.
- The draft write/reload/revert proof changed one private draft field and restored the captured original editable draft payload.
- This proof does not change backend, auth, control-plane, tenant, route, or deployment configuration.
- Public-route checks are route-render checks only; they do not intentionally call analytics-writing public-detail APIs directly.
- The proof was run against production because the branch Preview deployment was protected by Vercel SSO.
