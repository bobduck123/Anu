# Studio V3 P1 Foundation execution plan

Date: 2026-07-21
Branch: `feat/presence-studio-v3-p1-foundation`
Starting commit: `36204ca0c87e96f2fdbc61bf3fcad8606102e5f3`

## Outcome

Deliver the smallest reviewed backend foundation for durable owner-private Studio V3 state and conflict-safe existing-draft replacement, then prove the P1 three-Look and three-Room-Style range without changing public routes, publish behavior, production eligibility, or the V2 public renderer seam.

## Sequence

1. Freeze the backend scope and public/private boundaries.
2. Add the owner-private metadata and atomic existing-draft contracts with migration and backend tests.
3. Integrate durable metadata only when the backend contract is available; keep first-draft creation, publish, and server Visitor Preview out of scope.
4. Add Zine Archive, Film Strip / Selected Works, deterministic staged structural transformation, state-based Before/After, exact Cancel, and qualified restore.
5. Exercise the three Looks and three Room Styles on the same sanitized BBB fixture.
6. Run backend, compiler, type, build, focused Chromium, V2 regression, payload hygiene, diff, privacy, and request-ledger validation.
7. Capture the required desktop/mobile/public-invariance evidence and obtain independent aesthetic and no-merge reviews.
8. Commit only after an exact independent `VERDICT: MERGE`.

## Stop conditions

- The database work expands beyond one additive private-state table and one additive draft revision column.
- Existing-draft replacement cannot conflict before mutation or roll config and media state back together.
- Film Strip requires a second public renderer family.
- A public route, serializer, publish path, auth rule, tenant rule, production flag, or hosted record must change.
- GGM/private material, secrets, URLs, blobs, tokens, or raw auth identity enter Studio V3 private metadata or evidence.
- Required validation or independent review fails.

## Rollback boundary

Runtime rollback is limited to disabling the existing default-off local Studio V3 pilot. Code rollback removes the new private endpoints/service/table model and reverses the additive migration as documented in `MIGRATION_AND_ROLLBACK.md`; no public or published data transformation is planned.
