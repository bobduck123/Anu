# Client language map

Primary owner language changed while internal types, test identifiers, and persistence contracts remain stable.

| Internal/previous term | Owner-facing term | Reason |
| --- | --- | --- |
| Studio/editor cockpit | Your digital home / Studio | Establish purpose before tooling. |
| Chamber | Room | Familiar spatial model. |
| Object | Piece | Fits artwork, text, links, and calls to action. |
| Zone | Area | Explains placement without exposing layout schema. |
| Layout composition | Arrangement | Describes the owner's task. |
| Skin Lab | Look & Feel | Groups visual choices by intent. |
| Object Tuning | Edit Piece | Makes the selected-item relationship explicit. |
| Add Object | Add Piece | Consistent collection language. |
| Shadow depth | Density | Communicates visual weight rather than implementation. |
| Draft preview | Visitor Preview | Describes whose view is being checked. |
| Publish confirmation | Publish Review | Makes review precede the existing confirmation. |
| Save draft | Save changes | Clearer action while preserving draft semantics in explanatory copy. |
| Wild layout preset | Wild | Preserved in advanced controls for regression and expert continuity. |

Technical terms remain in code where renaming would increase risk or touch persisted contracts. The mapping is a presentation-layer translation, not a schema migration.
